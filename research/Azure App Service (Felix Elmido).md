- This is a fully managed platform as a service from Microsoft for building, hosting, and scaling web applications and APIs. It enables developers to deploy applications globally without managing servers, offering integrated tools for development, security, and continuous delivery.

### Core capabilities:

Azure App Service provides an enterprise-grade hosting environment for web apps, RESTful APIs, and mobile back ends. It abstracts away infrastructure management by automating provisioning, scaling, and patching. Developers can deploy directly from GitHub, Azure DevOps, or container registries. Built-in authentication integrates with Microsoft Entra ID and major social providers, while autoscaling ensures performance under variable workloads.

To create an Azure App Service we need a **App Service Plan**. 

What is an App Service Plan? 
It specifies our:
1. Operating System
2. Region
3. Instance Size
4. Scale Count
5. SKU / Pricing Tier

App Service Pricing Tier:
1. Shared (Free and Shared Tiers)
2. Dedicated (Basic, Standard, & Premium Tiers)
3. Isolated

Difference of Pricing Tiers:
1. CPU, Memory, Storage
2. Auto Scaling
3. Backups
4. Staging Slots
5. Custom Domains

**Scaling:**
Vertical Scaling = up or down
Horizontal = in or out
